# Claude Video Editing Flow -- Project CLAUDE.md

## What this project is

A selection-first video editing workflow. Danny drops a raw Riverside `.mp4`, Claude transcribes + scores + ranks every candidate segment, Danny ticks boxes in a markdown sheet, Claude renders. The target is short-form cuts (60s default) built by reading text, not scrubbing a timeline.

## The principle

**Selection is a human decision. Correction is a Claude decision.**

Humans belong at the candidate-sheet stage, where taste is cheap. Claude belongs at the render-iterate stage, where re-pulls cost seconds.

## Key rules

- **Terminal-first interaction.** Every picker / lock-in / verdict moment is a coloured table printed via `scripts/picker.py` / `lockin.py` / `verdict.py`. Markdown files are artefacts (git record, reproducibility) — never interfaces. The only non-terminal moment in the flow is QuickTime watching the rendered preview.
- **Never re-autonomously pick.** Candidates are generated by Claude; picks come from Danny as natural language in the terminal — combo letter, number list, free text.
- **One grade across the whole cut** -- `neutral_punch` preset. Never per-segment auto-grade.
- **No loudnorm.** Danny mixes audio in Riverside before export. Loudnorm here re-processes balanced audio and risks clipping.
- **30ms audio fades at every cut** -- prevents pops.
- **Framing rotation is auto-applied** -- 100% / 104% center / 100% / 104% right. Single-cam sources only. 4% scale change, no motion, no Ken Burns.
- **CRF 20 full render**, not `--preview` mode. Preview mode produces GOP artefacts at segment boundaries.
- **Captions are SRT**, not burned. Homebrew ffmpeg lacks libass. Ship SRT as companion file.
- **Target runtime has a 10% tolerance** -- 54-66s for a 60s target. Over-budget flags ★★ first as drop targets.

All rules live in `SELECTION-RULES.md`. Read once, rarely edit. Edit only when 3+ clips show the same correction pattern.

## File layout

```
SELECTION-RULES.md         # The SOP (persistent, rarely edited)
scripts/                   # render scripts -- per-clip ffmpeg pipelines
reference/                 # pipeline walk-throughs
sessions/                  # One markdown per processed clip
.claude/skills/            # Skill definition for intent routing
```

Per-clip working folders live OUTSIDE this repo (e.g. `../pod-test-claude/`). This project holds the rules, scripts and session logs only. Raw `.mp4` + `edit/` stay with the clip.

## Output contract

Every processed clip must produce:

1. `candidates.md` -- ranked picks with quote, duration, rationale, tier
2. `edl.json` -- built from ticked candidates, with framing rotation assigned by order
3. `preview.mp4` -- rendered cut, opened in QuickTime
4. `master.srt` -- captions from transcript + EDL offsets
5. A session log entry at `sessions/<date>-<clip-name>.md`

## When to edit SELECTION-RULES.md

- After 3+ clips show the same correction pattern -> encode the fix
- When output target changes (vertical Shorts, 30s micro-clips, 90s YouTube Shorts) -> add a separate framing rotation variant
- When a specific speaker has a signature style (e.g. Shubash needs more air before closing beats) -> per-speaker override

Don't edit for one-off exceptions. Per-clip tweaks belong in the EDL, not the rules.
